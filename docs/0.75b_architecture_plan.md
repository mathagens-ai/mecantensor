# Sub-1-Bit (0.75b) Tensor Engine Architecture

## Core Concept: The "Block-Palette" System
We are shifting from storing individual numbers to storing **topological patterns**.

### 1. The Data Structure
Instead of standard `float32` arrays, our tensor payload will consist of:
- **The Codebook (L1 Cache Resident):** A tiny 256-element array. Each element defines a 4x4 sparse pattern (16 weights).
- **The Payload (RAM):** A massive array of `uint8_t` (unsigned 8-bit integers). 
  - *Each 1-byte integer represents 16 weights.* 
  - *Total density: 8 bits / 16 weights = 0.5 bits per weight.*

### 2. Multi-Path Execution (The CPU Pipeline)
When a matrix multiplication (or Conv2D) is triggered, we do NOT use standard Math (`*` and `+`). We use Multi-Path Logic Gates:

**Path A: The Activation Pre-Compute**
When input data (like an image) arrives, we instantly pre-compute its dot product against all 256 possible patterns in the codebook. This creates a tiny "LUT" (Look-Up Table) of 256 exact answers.

**Path B: The Branchless Unpack**
We load the `uint8_t` weight payload into AVX2 256-bit registers (32 blocks at a time).
We use the `VPERMD` (Vector Permute) or `VPSHUFB` (Vector Shuffle) hardware instructions to map the 8-bit index directly to the pre-computed answer from Path A.

**Path C: The Accumulation**
We add the mapped answers together.
